Search Engine in Development

Jan 4th, 2012

Cloudkickr’s search engine is in Development through Ruby on Rails. Both Twitter and Groupon were built with Ruby on Rails. Our search engine architecture is designed for routekickr (the new site and app coming soon).

A search engine has three parts: Crawl, Index and Query. Crawl is similar to how your web browser gets you content to view while you’re surfing the internet. So the Crawl part of the search engine goes out and collects HTML pages. This data is indexed for query optimization, running the query part is what the end-user interacts with to find the relevent result, a data mapping between query and index .
